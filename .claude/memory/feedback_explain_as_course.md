description: When introducing new tools or workflows (spec-kit, GitHub Actions, etc.), explain step-by-step what each command does before running it

---

When using a tool or workflow for the first time, explain it as a course before executing. Walk through what the command does, what it creates, how it works, and why.

**Why:** User explicitly asked for this when learning spec-kit: "I want this to be a learning guide for me. So again, explain the command, what it creates, how it works, what it handles."

**How to apply:** Before running any new spec-kit command, GitHub Actions workflow, or unfamiliar tool, provide a "What this does (the course)" section explaining the purpose, inputs, outputs, and process. Don't skip this for subsequent uses of the same command — the user confirmed they want it every time for new steps.
