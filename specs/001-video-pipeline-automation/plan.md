# Implementation Plan: Video Pipeline Automation

**Branch**: `001-video-pipeline-automation` | **Date**: 2026-05-05 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/001-video-pipeline-automation/spec.md`

## Summary

Build a Flutter desktop app (Windows) that automates the video production team's post-shoot workflow: detect SD cards, transfer files via robocopy with resume capability, compress via HandBrakeCLI with custom presets, and notify via Slack. The app uses a job queue model where each job is independently configurable (source, destination, preset, auto-chain).

## Technical Context

**Language/Version**: Dart 3.x (via Flutter SDK stable)
**Framework**: Flutter 3.x (desktop, Windows target)
**Primary Dependencies**: win32, device_manager, drift, sqflite_common_ffi, dio
**Storage**: SQLite via drift ORM (local database for job queue and state)
**Testing**: flutter_test, integration_test
**Target Platform**: Windows 11 (x64)
**Project Type**: Desktop application
**Performance Goals**: Real-time progress updates (sub-second UI refresh), file verification on every transfer
**Constraints**: Single machine, single user, offline-capable (pipeline runs without internet; Slack is best-effort)
**Scale/Scope**: 1-10 jobs per session, 1-20 files per job (50-100 GB each)

### Key Design Decisions

1. **Reactive UI via Drift `.watch()`**: The job queue UI subscribes to database queries. When a service updates a job's status or progress, the UI rebuilds automatically. No manual state synchronization needed.

2. **One job processes at a time**: Sequential queue processing. This avoids disk I/O contention (one robocopy or one HandBrake at a time). Simple and predictable.

3. **Progress tracking strategy**:
   - Transfer: file-level completion (files done / total files) + intra-file progress via destination file size polling
   - Compression: HandBrakeCLI stdout percentage parsing in real-time

4. **Auto-chain implementation**: When a transfer job with `auto_chain = true` completes, the `JobQueueService` creates a new compression job with the same file list and queues it next.

5. **Favorites system**: Simple database table. UI shows a dropdown of saved paths with a "Save as Favorite" button next to any folder picker.
